# Capstone

캡스톤 디자인 수업용

## 프로젝트 개요
실습실 PC 사용 관리 시스템은 학생이 모바일 앱 또는 PC 앱에서 로그인하고, PC 앱에 표시된 인증 코드를 모바일 앱에서 입력하여

PC 사용을 인증하며, 사용 시간 확인 및 연장 신청을 수행하고, 관리자가 PC 앱에서 회원과 PC 사용 현황을 관리할 수 있도록 하는 시스템이다.

## 화면 구조

### 모바일 앱

#### 로그인 전
- 로그인
- 회원가입
- 학번 찾기
- 비밀번호 찾기

#### 인증 화면
- PC 인증 코드 입력

#### 홈 화면
- 비밀번호 변경
- 회원탈퇴
- 로그아웃
- 사용 시간 확인
- 사용 시간 연장 신청
- 알림 확인

### PC 앱

#### 로그인 화면
- 일반 사용자 로그인
- 관리자 계정 접속

#### 일반 사용자 메인 화면
- 인증 코드 표시
- 사용 시간 확인
- 사용 시간 연장 신청
- 알림 확인
- 사용 시간 종료 시 자동 종료

#### 관리자 로그인 화면
- 관리자 로그인

#### 관리자용 화면
- 사용자 리스트 표시
- 선택한 사용자의 현재 세션 정보 표시
- 선택한 사용자의 세션 히스토리 표시
- 악성 사용자 제한
- 특정 PC 강제 종료 명령

# 학생 이벤트 시나리오

## 1. 회원가입하기
- **유스케이스명**: 회원가입
- **액터**: 학생
- **선행조건**
  - 학생은 모바일 앱에 접속할 수 있어야 한다.
  - 아직 회원가입이 완료되지 않은 상태여야 한다.
- **선행입력**
  - 학번
  - 이름
  - 이메일
  - 비밀번호
- **이벤트 흐름**
  1. 학생이 모바일 앱의 로그인 전 화면에서 회원가입 메뉴를 선택한다.
  2. 학생이 학번, 이름, 이메일, 비밀번호를 입력한다.

  1. 관리자가 관리자용 PC 앱 화면에 접속한다.
  2. 시스템은 사용자 리스트를 표시한다.
  3. 관리자가 특정 사용자를 선택한다.
  4. 시스템은 해당 사용자의 현재 세션 정보와 세션 히스토리를 함께 표시한다.
  5. 관리자는 필요 시 사용자, PC, 날짜 조건으로 검색 또는 필터링한다.
- **후행조건**
  - 관리자가 현재 세션 정보와 과거 사용 이력을 확인한다.
- **후행출력**
  - 사용자 목록
  - 현재 세션 정보
  - 세션 히스토리
- **제약사항**
  - 관리자 권한이 필요하다.

## 4. 악성 사용자 제한 및 PC 제어하기
- **유스케이스명**: 악성 사용자 제한 및 PC 제어
- **액터**: 관리자
- **선행조건**
  - 관리자는 관리자 권한으로 로그인한 상태여야 한다.
  - 대상 사용자 또는 대상 PC가 존재해야 한다.
- **선행입력**
  - 대상 사용자 선택 또는 대상 PC 선택
- **이벤트 흐름**
  1. 관리자가 관리자용 PC 앱 화면에서 악성 사용자를 선택하거나 특정 PC를 선택한다.
  2. 시스템은 선택한 대상의 상세 정보를 표시한다.
  3. 관리자는 사용자 제한 또는 PC 강제 종료 중 하나를 선택한다.
  4. 사용자 제한을 선택한 경우, 시스템은 해당 사용자의 사용 권한을 제한 상태로 변경한다.
  5. PC 강제 종료를 선택한 경우, 시스템은 해당 PC 앱에 종료 명령을 전송한다.
  6. PC 앱은 종료 명령을 수신하고 운영체제 종료 명령을 실행한다.
  7. 시스템은 필요한 경우 세션 상태를 종료로 갱신한다.
- **후행조건**
  - 대상 사용자가 제한 상태가 되거나 대상 PC가 종료된다.
- **후행출력**
  - 사용자 제한 결과 메시지 또는 PC 종료 결과 메시지
- **제약사항**
  - 관리자 권한이 필요하다.
  - 이미 제한된 사용자에게는 중복 제한을 적용하지 않는다.
  - PC 강제 종료는 대상 PC가 온라인 상태여야 한다.
