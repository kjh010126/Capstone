# Capstone
캡스톤 디자인 수업용

# 기능적 요구사항

| 요구사항 ID | 요구사항 명 | 개요 | 연관 요구사항 | 상세 설명 | 유형 | 중요도 | 난이도 |
|---|---|---|---|---|---|---|---|
| FR-01 | 회원 가입 API | 사용자 회원 등록 | 없음 | 이메일, 비밀번호, 이름 등을 입력받아 회원가입을 처리하고, 중복 계정을 검증한 뒤 DB에 저장한다. | 회원 관리 | 높음 | 낮음 |
| FR-02 | 회원 로그인 API | 사용자 인증 수행 | FR-01 | 가입된 사용자가 이메일과 비밀번호로 로그인하며, 성공 시 JWT 등 인증 토큰을 발급한다. | 인증 | 높음 | 낮음 |
| FR-03 | 회원 탈퇴 API | 회원 계정 삭제 | FR-02 | 로그인한 사용자가 본인 계정을 삭제할 수 있으며, 관련 데이터는 비활성화 또는 삭제 처리한다. | 회원 관리 | 중간 | 낮음 |
| FR-04 | 회원 CRUD | 회원 정보 관리 | FR-01 | 관리자 또는 시스템이 회원 정보 조회, 수정, 삭제 등 기본 CRUD 기능을 수행할 수 있다. | 회원 관리 | 높음 | 중간 |
| FR-05 | 컴퓨터 사용 신청 API | PC 사용 신청 | FR-02 | 로그인한 사용자가 특정 PC 사용을 신청하면, 사용 시작 및 종료 시간을 포함한 사용 세션을 생성한다. | PC 사용 관리 | 높음 | 중간 |
| FR-06 | 사용 연장 신청 API | 사용 시간 연장 | FR-05 | 사용자가 현재 사용 중인 PC의 사용 시간을 연장 신청할 수 있으며, 서버는 연장 가능 여부를 검증한다. | PC 사용 관리 | 높음 | 중간 |
| FR-07 | 관리자용 API | 관리자 기능 제공 | FR-04, FR-05 | 관리자는 전체 회원 조회, PC 사용 현황 조회, 사용 승인 및 제한 등 관리 기능을 수행할 수 있다. | 관리자 기능 | 높음 | 중간 |
| FR-08 | 웹소켓 연결 API | 실시간 상태 관리 | FR-05 | 클라이언트와 서버가 웹소켓으로 연결되어 PC 상태, 연장 요청, 알림 등을 실시간으로 주고받는다. | 실시간 통신 | 높음 | 높음 |

# 비기능적 요구사항

| 요구사항 ID | 요구사항 명 | 개요 | 연관 요구사항 | 상세 설명 | 유형 | 중요도 | 난이도 |
|---|---|---|---|---|---|---|---|
| NFR-01 | 인증 보안 | 사용자 인증 보호 | FR-02 | 로그인 시 JWT 기반 인증을 적용하고, 비밀번호는 Bcrypt 등으로 암호화하여 저장한다. | 보안 | 높음 | 중간 |
| NFR-02 | API 응답 속도 | 빠른 응답 성능 | 전체 | 모든 API는 평균 응답 시간을 1초 이내로 유지하도록 설계한다. | 성능 | 높음 | 중간 |
| NFR-03 | 동시 접속 처리 | 다중 사용자 처리 | FR-05, FR-08 | 여러 사용자가 동시에 PC 사용을 신청하더라도 서버가 안정적으로 요청을 처리해야 한다. | 확장성 | 높음 | 중간 |
| NFR-04 | 실시간 데이터 처리 | 웹소켓 안정성 | FR-08 | 웹소켓 연결은 안정적으로 유지되어야 하며, 연결이 끊길 경우 재연결 로직을 지원해야 한다. | 신뢰성 | 중간 | 높음 |
| NFR-05 | 데이터 무결성 | 데이터 정확성 보장 | FR-05, FR-06 | 동일 PC에 대한 중복 사용 신청이 발생하지 않도록 검증 및 트랜잭션 처리를 적용한다. | 데이터 | 높음 | 중간 |
| NFR-06 | 로그 및 모니터링 | 운영 상태 관리 | 전체 | API 요청, 오류 로그, 시스템 상태를 기록하고 모니터링할 수 있어야 한다. | 운영 | 중간 | 낮음 |
