|---|---|---|---|---|---|---|---|
| FR-01 | 서버 및 DB / 모바일 앱 | 회원가입 기능 | 사용자가 계정을 생성할 수 있어야 함 | 없음 | 사용자는 모바일 앱에서 학번, 이름, 비밀번호, 이메일 등의 정보를 입력하여 회원가입을 수행할 수 있어야 하며, 서버는 비밀번호 유효성 검사를 한 뒤 DB에 저장해야 한다. | 높음 | 낮음 |
| FR-02 | 서버 및 DB / 모바일 앱 | 로그인 기능 | 사용자가 시스템에 로그인할 수 있어야 함 | FR-01 | 사용자는 모바일 앱에서 학번과 비밀번호를 이용해 로그인할 수 있어야 하며, 서버는 로그인 성공 시 인증 토큰을 발급해야 한다. | 높음 | 낮음 |
| FR-03 | 모바일 앱 | 자동로그인 기능 | 사용자가 재로그인 없이 앱을 사용할 수 있어야 함 | FR-02 | 모바일 앱은 저장된 인증 정보를 활용하여 앱 재실행 시 자동로그인을 지원해야 한다. | 중간 | 낮음 |
| FR-04 | 모바일 앱 | 로그아웃 기능 | 사용자가 로그인 세션을 종료할 수 있어야 함 | FR-02 | 사용자는 모바일 앱에서 로그아웃할 수 있어야 하며, 앱은 저장된 인증 정보를 삭제해야 한다. | 중간 | 낮음 |
| FR-05 | 서버 및 DB / 모바일 앱 | 회원탈퇴 기능 | 사용자가 계정을 삭제할 수 있어야 함 | FR-02 | 로그인한 사용자는 모바일 앱에서 본인 계정을 탈퇴할 수 있어야 하며, 서버는 관련 사용자 정보를 비활성화 또는 삭제 처리해야 한다. | 중간 | 낮음 |
| FR-06 | 서버 및 DB / 관리자 | 회원 관리 기능 | 관리자가 회원 정보를 관리할 수 있어야 함 | FR-01, FR-02 | 서버는 관리자 기능을 통해 회원 조회, 수정, 삭제 등의 회원 관리 기능을 제공해야 한다. | 높음 | 중간 |
| FR-07 | 서버 및 DB / 모바일 앱 | 비밀번호 찾기, 변경 기능 | 사용자가 비밀번호를 찾기 및 변경이 가능해야함 | FR-02, FR-03, FR-06 | 비밀번호를 찾기 및 변경을 할 때는 학번과 이메일를 입력하여 인증한다 | 높음 | 중간 |
| FR-08 | 서버 및 DB / PC 앱 | 인증 코드 발급 기능 | PC 사용을 위한 인증 코드를 생성해야 함 | FR-02 | PC가 켜지면 PC 앱은 서버에 인증 코드 발급을 요청해야 하며, 서버는 해당 PC에 대한 일회용 숫자 코드를 생성하여 반환해야 한다. | 높음 | 중간 |
| FR-09 | PC 앱 | 인증 코드 표시 기능 | 발급된 인증 코드를 화면에 보여줘야 함 | FR-07 | PC 앱은 서버에서 전달받은 숫자 인증 코드를 화면에 표시하여 사용자가 확인할 수 있도록 해야 한다. | 높음 | 낮음 |
| FR-10 | 서버 및 DB / 모바일 앱 | 코드 인증 기능 | 사용자가 숫자 코드를 입력하여 PC 사용을 인증할 수 있어야 함 | FR-07, FR-08 | 사용자는 모바일 앱에서 PC 화면에 표시된 숫자 코드를 입력할 수 있어야 하며, 서버는 코드가 유효한 경우 해당 PC 사용을 승인해야 한다. | 높음 | 중간 |
| FR-11 | 서버 및 DB | 사용 세션 생성 및 관리 기능 | 인증된 사용자에 대한 PC 사용 세션을 관리해야 함 | FR-09 | 서버는 코드 인증이 완료되면 사용자와 PC를 연결하는 사용 세션을 생성하고, 시작 시간, 종료 예정 시간, 상태를 저장 및 관리해야 한다. | 높음 | 중간 |
| FR-12 | PC 앱 | 인증 상태 반영 기능 | 인증 완료 후 PC 사용 가능 상태로 전환해야 함 | FR-09, FR-10 | 서버에서 인증 완료가 확인되면 PC 앱은 인증 대기 상태에서 사용 가능 상태로 전환되어야 한다. | 높음 | 중간 |
| FR-13 | PC 앱 | 사용 시간 표시 기능 | 현재 사용 상태와 남은 시간을 보여줘야 함 | FR-10 | PC 앱은 현재 사용자 인증 상태와 남은 사용 시간을 화면에 표시해야 한다. | 중간 | 낮음 |
| FR-14 | 서버 및 DB / PC 앱 / 모바일 앱 | 알림 기능 | 사용 종료 전 알림을 제공해야 함 | FR-10 | 서버는 사용 종료 10분 전에 PC 앱과 모바일 앱에 알림을 전송해야 하며, 각 앱은 이를 사용자에게 표시해야 한다. | 높음 | 중간 |
| FR-15 | 서버 및 DB / 모바일 앱 / PC 앱 | 사용 연장 신청 기능 | 사용자가 사용 시간을 연장할 수 있어야 함 | FR-10, FR-13 | 사용자는 모바일 앱 또는 PC 앱에서 사용 연장을 신청할 수 있어야 하며, 서버는 연장 가능 여부를 검증한 뒤 종료 시간을 갱신해야 한다. | 높음 | 중간 |
| FR-16 | 서버 및 DB | 연장 검증 기능 | 연장 가능 여부를 판단해야 함 | FR-14 | 서버는 남은 시간, 현재 세션 상태, 정책 조건 등을 기준으로 연장 가능 여부를 판단해야 한다. | 높음 | 중간 |
| FR-17 | PC 앱 / 모바일 앱 | 알림 표시 기능 | 종료 전 알림 및 연장 안내를 화면에 표시해야 함 | FR-13 | 앱은 남은 시간이 10분 이하일 때 서버로부터 전달받은 종료 예정 및 연장 가능 여부를 알리는 메시지를 사용자에게 표시해야 한다. | 높음 | 낮음 |
| FR-18 | PC 앱 | 자동 종료 기능 | 사용 시간이 끝나면 PC를 종료해야 함 | FR-10, FR-15 | 연장되지 않은 상태에서 사용 시간이 만료되면 PC 앱은 운영체제 종료 명령을 실행하여 PC를 자동 종료해야 한다. | 높음 | 높음 |
| FR-19 | 서버 및 DB / 관리자 | 관리자 조회 기능 | 관리자가 전체 현황을 확인할 수 있어야 함 | FR-06, FR-10 | 관리자는 전체 회원 목록, PC 사용 현황, 현재 세션 상태 등을 조회할 수 있어야 한다. | 높음 | 중간 |
| FR-20 | 서버 및 DB / 관리자 | 관리자 제어 기능 | 관리자가 시스템을 통제할 수 있어야 함 | FR-19 | 관리자는 특정 사용자 사용 제한, 특정 PC 사용 상태 확인, 필요 시 강제 종료 또는 제한 조치를 수행할 수 있어야 한다. | 중간 | 중간 |
| FR-21 | 서버 및 DB / PC 앱 / 모바일 앱 | 웹소켓 실시간 통신 기능 | 상태 및 알림을 실시간으로 주고받아야 함 | FR-10, FR-13, FR-14 | 서버와 클라이언트는 웹소켓으로 연결되어 인증 상태, 남은 시간, 연장 결과, 종료 알림 등을 실시간으로 주고받아야 한다. | 높음 | 높음 |
