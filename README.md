# 학생 이벤트 시나리오

## 1. 회원가입하기

- **액터**: 학생

- **사전조건**
  - 학생은 모바일 앱에 접속할 수 있어야 한다.
  - 아직 회원가입이 완료되지 않은 상태여야 한다.

- **기본 흐름**
  1. 학생이 모바일 앱에서 회원가입 화면에 접속한다.
  2. 학생이 학번, 이름, 이메일, 비밀번호를 입력한다.
  3. 학생이 회원가입 버튼을 누른다.
  4. 시스템은 입력값 형식을 검증한다.
  5. 시스템은 학번 및 이메일 중복 여부를 확인한다.
  6. 시스템은 비밀번호 유효성 검사를 수행한다.
  7. 검증이 완료되면 시스템은 회원 정보를 DB에 저장한다.
  8. 시스템은 회원가입 완료 메시지를 표시한다.

- **예외 흐름**
  - 5-1. 학번 또는 이메일이 이미 존재하면 회원가입을 거부한다.
  - 6-1. 비밀번호 형식이 조건에 맞지 않으면 오류 메시지를 표시한다.

- **사후조건**
  - 학생 계정이 생성된다.

---

## 2. 로그인하기

- **액터**: 학생

- **사전조건**
  - 학생은 회원가입을 완료한 상태여야 한다.

- **기본 흐름**
  1. 학생이 모바일 앱에서 로그인 화면에 접속한다.
  2. 학생이 학번과 비밀번호를 입력한다.
  3. 학생이 로그인 버튼을 누른다.
  4. 시스템은 입력된 정보를 DB와 비교하여 인증한다.
  5. 인증이 성공하면 시스템은 인증 토큰을 발급한다.
  6. 모바일 앱은 인증 토큰을 저장한다.
  7. 학생은 메인 화면으로 이동한다.

- **예외 흐름**
  - 4-1. 학번 또는 비밀번호가 일치하지 않으면 로그인에 실패한다.
  - 4-2. 서버 오류가 발생하면 로그인 실패 메시지를 표시한다.

- **사후조건**
  - 학생이 로그인 상태가 된다.

---

## 3. 자동로그인하기

- **액터**: 학생

- **사전조건**
  - 학생이 이전에 로그인한 적이 있어야 한다.
  - 모바일 앱에 인증 정보가 저장되어 있어야 한다.

- **기본 흐름**
  1. 학생이 모바일 앱을 실행한다.
  2. 앱은 저장된 인증 토큰을 확인한다.
  3. 시스템은 토큰의 유효성을 검증한다.
  4. 토큰이 유효하면 학생은 자동으로 메인 화면에 진입한다.

- **예외 흐름**
  - 2-1. 저장된 인증 정보가 없으면 자동로그인을 수행하지 않는다.
  - 3-1. 저장된 토큰이 만료되었으면 로그인 화면으로 이동한다.

- **사후조건**
