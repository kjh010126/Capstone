# Capstone

캡스톤 디자인 수업용

## 프로젝트 개요
실습실 PC 사용 관리 시스템은 학생이 모바일 앱으로 PC 사용을 인증하고, 사용 시간 확인 및 연장 신청을 수행하며, 관리자가 회원과 PC 사용 현황을 관리할 수 있도록 하는 시스템이다.

## 모바일 앱 화면 구조

### 로그인 전
- 로그인
- 회원가입
- 학번 찾기
- 비밀번호 찾기

### 로그인 후 메인
- PC 인증
- 사용 시간 확인
- 사용 시간 연장 신청
- 알림 확인
- 내 정보

### 내 정보 / 계정관리
- 내 정보 조회
- 비밀번호 변경
- 회원탈퇴
- 로그아웃

# 학생 이벤트 시나리오

## 1. 회원가입하기

- **액터**: 학생

- **사전조건**
  - 학생은 모바일 앱에 접속할 수 있어야 한다.
  - 아직 회원가입이 완료되지 않은 상태여야 한다.

- **선행입력**
  - 학번
  - 이름
  - 이메일
  - 비밀번호

- **기본 흐름**
  1. 학생이 모바일 앱에서 회원가입 화면에 접속한다.
  2. 학생이 학번, 이름, 이메일, 비밀번호를 입력한다.
  3. 학생이 회원가입 버튼을 누른다.
  4. 시스템은 입력값 형식을 검증한다.
  5. 시스템은 학번 및 이메일 중복 여부를 확인한다.
  6. 시스템은 비밀번호 유효성 검사를 수행한다.
  7. 검증이 완료되면 시스템은 회원 정보를 DB에 저장한다.
  8. 시스템은 회원가입 완료 메시지를 표시한다.

- **예외 흐름**
  - 5-1. 학번 또는 이메일이 이미 존재하면 회원가입을 거부한다.
  - 6-1. 비밀번호 형식이 조건에 맞지 않으면 오류 메시지를 표시한다.

- **사후조건**
  - 학생 계정이 생성된다.

- **후행출력**
  - 회원가입 완료 메시지

- **제약사항**
  - 학번 및 이메일은 중복될 수 없다.
  - 비밀번호는 보안 정책을 만족해야 한다.

## 2. 로그인하기

- **액터**: 학생

- **사전조건**
  - 학생은 회원가입을 완료한 상태여야 한다.

- **선행입력**
  - 학번
  - 비밀번호

- **기본 흐름**
  1. 학생이 모바일 앱에서 로그인 화면에 접속한다.
  2. 학생이 학번과 비밀번호를 입력한다.
  3. 학생이 로그인 버튼을 누른다.
  4. 시스템은 입력된 정보를 DB와 비교하여 인증한다.
  5. 인증이 성공하면 시스템은 인증 토큰을 발급한다.
  6. 모바일 앱은 인증 토큰을 저장한다.
  7. 학생은 메인 화면으로 이동한다.

- **예외 흐름**
  - 4-1. 학번 또는 비밀번호가 일치하지 않으면 로그인에 실패한다.
  - 4-2. 서버 오류가 발생하면 로그인 실패 메시지를 표시한다.

  - 종료 성공 또는 실패 메시지

- **제약사항**
  - 관리자 권한이 필요하다.
  - PC는 온라인 상태여야 한다.

## 5. 사용 이력 조회하기

- **액터**: 관리자

- **사전조건**
  - 관리자는 로그인 상태여야 한다.

- **선행입력**
  - 조회 조건 (날짜, 사용자, PC 등)

- **기본 흐름**
  1. 관리자가 사용 이력 조회 메뉴를 선택한다.
  2. 시스템은 날짜별, 사용자별, PC별 이력 목록을 표시한다.
  3. 관리자는 원하는 조건으로 검색 또는 필터링한다.
  4. 시스템은 검색 결과를 출력한다.

- **예외 흐름**
  - 4-1. 검색 조건에 해당하는 이력이 없으면 빈 결과를 표시한다.

- **사후조건**
  - 관리자가 과거 사용 기록을 확인한다.

- **후행출력**
  - 사용 기록 목록

- **제약사항**
  - 관리자 권한이 필요하다..
