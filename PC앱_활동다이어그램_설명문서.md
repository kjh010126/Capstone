# PC 앱 활동 다이어그램 설명 문서

## 1. 문서 개요
본 문서는 실습실 PC 사용 관리 시스템의 **PC 앱 활동 다이어그램**에 대한 설명 문서이다.  
현재 PC 앱 활동 다이어그램은 변경된 요구사항을 반영하여, 일반 사용자 PC 로그인 기능을 제거하고 **모바일 앱에서 표시된 인증 코드를 PC 앱에서 입력하는 구조**를 중심으로 재구성하였다.

또한 활동 다이어그램은 PC 앱 담당 범위만 표현하도록 정리하였으며, 모바일 앱 내부 흐름은 제외하고 **PC 앱이 실제로 수행하거나 반영하는 동작**을 중심으로 구성하였다.

본 문서는 다음 다섯 개의 PC 앱 활동 다이어그램을 설명한다.

1. PC 앱 코드 인증 및 사용 시작
2. 사용 시간 연장 신청 및 자동 로그아웃
3. 관리자 로그인
4. 관리자 조회 / 제한 / 강제 종료
5. 자동 종료

---

## 2. 활동 다이어그램 구성 원칙

### 2.1 PC 앱 담당 범위만 표현한다
활동 다이어그램은 시스템 전체가 아니라 **PC 앱 담당 기능**을 기준으로 작성하였다.  
따라서 모바일 앱 내부 화면 전환이나 서버 내부 세부 구현 로직은 최소화하고, PC 앱이 요청을 보내거나 상태를 반영하는 흐름에 집중하였다.

### 2.2 하나의 다이어그램은 하나의 주요 흐름만 표현한다
기존에는 로그인, 인증, 연장, 종료 흐름이 섞일 수 있었지만, 현재는 각 주요 기능을 개별 활동 다이어그램으로 분리하였다.  
이를 통해 각 다이어그램이 무엇을 설명하는지 명확하게 하고, 읽기 쉽도록 정리하였다.

### 2.3 스윔레인은 실제 참여 주체만 포함한다
각 활동 다이어그램은 해당 흐름에 직접 참여하는 주체만 스윔레인으로 포함한다.  
예를 들어 관리자 기능에서는 `관리자 | PC 앱 | 서버 | 대상 PC 앱`을 사용하고, 자동 종료에서는 `서버 | PC 앱 | 운영체제`를 사용한다.

### 2.4 변경된 요구사항을 반영하였다
변경된 요구사항은 다음과 같다.

- 일반 사용자는 PC 앱에서 로그인하지 않는다.
- 일반 사용자는 모바일 앱에 표시된 인증 코드를 PC 앱에 입력하여 인증한다.
- 연장 신청을 하지 않으면 자동 로그아웃된다.
- 사용 시간이 종료되면 PC 앱은 자동 종료를 수행한다.

이에 따라 기존 활동 다이어그램 중 **일반 사용자 로그인 및 인증 코드 표시** 흐름은 제거되고, **코드 인증 및 사용 시작** 흐름 중심으로 재구성되었다.

## 3. 전체 활동 다이어그램 구성
현재 PC 앱 활동 다이어그램은 다음 다섯 개로 구성된다.

### 3.1 PC 앱 코드 인증 및 사용 시작
학생이 모바일 앱에서 인증 코드를 확인한 뒤 PC 앱에 입력하고, 서버가 이를 검증하여 세션을 생성한 후 PC 앱이 사용 가능 상태로 전환되는 흐름을 나타낸다.

### 3.2 사용 시간 연장 신청 및 자동 로그아웃
학생이 연장 신청을 수행하거나 수행하지 않는 경우를 분기하여, 연장 성공/실패와 자동 로그아웃 흐름을 함께 표현한다.

### 3.3 관리자 로그인
관리자가 관리자 계정 접속 메뉴를 통해 관리자 로그인 화면으로 진입하고, 인증 성공 시 관리자용 화면으로 이동하는 흐름을 나타낸다.

### 3.4 관리자 조회 / 제한 / 강제 종료
관리자가 사용자 목록과 세션 정보를 조회한 뒤, 조회만 수행하거나 사용자 제한 또는 PC 강제 종료를 선택하는 흐름을 나타낸다.

### 3.5 자동 종료
서버가 전달한 세션 종료 시각을 기준으로 PC 앱이 타이머를 수행하고, 종료 시점에 운영체제 종료 명령을 실행하는 흐름을 나타낸다.

## 4. 활동 다이어그램별 상세 설명

## 4.1 PC 앱 코드 인증 및 사용 시작
### 목적
모바일 앱에 표시된 인증 코드를 학생이 PC 앱에 입력하고, 인증 성공 시 PC 사용 가능 상태로 전환되는 과정을 표현한다.

### 스윔레인
- 학생
- PC 앱
- 서버

### 주요 흐름
1. 학생이 모바일 앱에 표시된 인증 코드를 확인한다.
2. 학생이 PC 앱의 코드 인증 화면에 접속한다.
3. 학생이 인증 코드를 입력하고 인증 버튼을 누른다.
4. PC 앱은 서버에 코드 검증 요청을 전송한다.
5. 서버는 인증 코드 유효성을 검증한다.
6. 코드가 유효하면 서버는 사용자-PC 세션을 생성한다.
7. 서버는 PC 앱에 인증 완료 상태를 전달한다.
8. PC 앱은 사용 가능 상태로 전환된다.
9. PC 앱은 사용 시간 표시를 시작하고 일반 사용자 메인 화면을 갱신한다.

### 분기 흐름
- 인증 실패: PC 앱은 인증 실패 메시지를 표시하고 흐름을 종료한다.
- 인증 성공: 세션 생성 후 메인 화면으로 전환된다.

### 설명
기존에는 PC 앱이 인증 코드를 표시하고 모바일 앱이 입력하는 구조였지만, 현재는 반대로 모바일 앱이 코드를 표시하고 PC 앱이 입력받는다.  
따라서 PC 앱 활동 다이어그램에서는 일반 사용자 로그인 흐름을 제거하고, **코드 입력 이후의 인증 반영 과정**을 중심으로 표현하였다.

---

## 4.2 사용 시간 연장 신청 및 자동 로그아웃
### 목적
학생이 사용 시간 연장 신청을 수행하는 경우와 수행하지 않는 경우를 함께 표현하여, 연장 처리와 자동 로그아웃 흐름을 설명한다.

### 스윔레인
- 학생
- PC 앱
- 서버

### 주요 흐름
1. 시스템은 먼저 연장 신청 수행 여부를 판단한다.
2. 학생이 연장 신청 버튼을 누르면 PC 앱은 서버에 연장 요청을 전송한다.
3. 서버는 현재 세션을 조회한다.
4. 서버는 연장 가능 여부를 검증한다.
5. 연장이 가능하면 서버는 종료 예정 시간을 갱신한다.
6. PC 앱은 갱신된 남은 시간을 표시하고 연장 완료 메시지를 표시한다.

### 분기 흐름
- 연장 신청 안 함: PC 앱은 자동 로그아웃을 처리하고 종료한다.
- 연장 불가: PC 앱은 연장 불가 메시지를 표시하고 종료한다.
- 연장 가능: 갱신된 종료 시간과 남은 시간이 반영된다.

### 설명
기존 활동 다이어그램에서는 연장 신청만 표현했지만, 변경된 요구사항에 따라 **연장 신청 미수행 시 자동 로그아웃**을 함께 포함하였다.  
따라서 이 다이어그램은 단순 연장 요청 흐름이 아니라, 세션 연장 정책 전체를 보여주는 역할을 한다.

---

## 4.3 관리자 로그인
### 목적
관리자가 PC 앱의 관리자 로그인 화면을 통해 관리자용 화면으로 진입하는 과정을 표현한다.

### 스윔레인
- 관리자
- PC 앱
- 서버

### 주요 흐름
1. 관리자가 관리자 계정 접속을 선택한다.
2. PC 앱은 관리자 로그인 화면을 표시한다.
3. 관리자가 관리자 ID와 비밀번호를 입력한다.
4. 관리자가 로그인 버튼을 클릭한다.
5. PC 앱은 서버에 로그인 요청을 전송한다.
6. 서버는 관리자 계정을 검증한다.
7. 로그인 성공 시 PC 앱은 관리자용 화면을 표시한다.

### 분기 흐름
- 로그인 실패: PC 앱은 로그인 실패 메시지를 표시하고 종료한다.
- 로그인 성공: 관리자용 화면으로 이동한다.

### 설명
현재 PC 앱에서는 일반 사용자 로그인 기능이 제거되었으므로, 로그인 화면은 실질적으로 **관리자 전용 진입 흐름**만 가진다.  
따라서 활동 다이어그램도 관리자 로그인 중심으로 단순화하였다.

---

## 4.4 관리자 조회 / 제한 / 강제 종료
### 목적
관리자가 관리자용 화면에서 사용자와 세션 정보를 조회하고, 필요 시 사용자 제한 또는 특정 PC 강제 종료를 수행하는 흐름을 표현한다.

### 스윔레인
- 관리자
- PC 앱
- 서버
- 대상 PC 앱

### 주요 흐름
1. 관리자가 관리자용 화면에 접속한다.
2. PC 앱은 사용자 목록 조회 요청을 전송한다.
3. 서버는 사용자 목록을 반환한다.
4. PC 앱은 사용자 목록을 표시한다.
5. 관리자가 특정 사용자를 선택한다.
6. PC 앱은 세션 정보 조회 요청을 전송한다.
7. 서버는 현재 세션과 세션 히스토리를 반환한다.
8. PC 앱은 조회 결과를 표시한다.
9. 이후 시스템은 수행 작업을 선택한다.

### 분기 흐름
- 조회만 수행: 결과 확인 후 종료한다.
- 사용자 제한: 서버는 계정 제한 상태를 변경하고, PC 앱은 제한 결과를 표시한다.
- PC 강제 종료: 서버는 대상 PC 앱에 종료 명령을 전달하고, 대상 PC 앱은 종료 명령을 실행한다. 이후 서버는 세션 상태를 종료로 갱신한다.

### 설명
이 다이어그램은 관리자 기능 중 **조회와 제어를 하나의 흐름으로 통합**하여 표현한 것이다.  
사용자 조회, 세션 조회, 제한, 강제 종료가 모두 관리자용 화면에서 이어지는 작업이므로, 관리 기능의 실제 사용 흐름을 잘 보여준다.

## 4.5 자동 종료
### 목적
세션 종료 시각에 도달했을 때 PC 앱이 자동으로 운영체제 종료 명령을 수행하는 흐름을 표현한다.

### 스윔레인
- 서버
- PC 앱
- 운영체제

### 주요 흐름
1. 서버가 세션 종료 시각을 PC 앱에 전달한다.
2. PC 앱은 해당 시각을 기준으로 로컬 타이머를 시작한다.
3. PC 앱은 종료 시각 도달 여부를 반복적으로 판단한다.
4. 종료 시각에 도달하지 않았으면 남은 시간을 계속 표시한다.
5. 종료 시각에 도달하면 PC 앱은 종료 명령을 실행한다.
6. 운영체제는 PC를 종료한다.

### 분기 흐름
- 미도달: 남은 시간 표시 후 다시 종료 시각 도달 여부를 확인한다.
- 도달: 종료 명령을 실행하고 종료한다.

### 설명
자동 종료는 사용 시간 만료 이후의 마지막 시스템 동작이므로, 별도의 활동 다이어그램으로 독립적으로 표현하였다.  
이 다이어그램은 세션 종료 시각 전달, 로컬 타이머 수행, 운영체제 종료 명령 실행의 흐름을 명확히 보여준다.

## 5. 기존 활동 다이어그램과의 차이
기존 PC 앱 활동 다이어그램은 다음과 같은 흐름을 포함하고 있었다.

- 일반 사용자 로그인 및 인증 코드 표시
- 인증 완료 상태 반영 및 사용 시작
- 사용 시간 연장 신청
- 관리자 로그인
- 관리자 조회 / 제한 / 강제 종료
- 자동 종료

하지만 현재 요구사항 변경에 따라 다음과 같이 수정되었다.

- **일반 사용자 로그인 및 인증 코드 표시** 흐름은 제거하였다.
- 일반 사용자 관련 흐름은 **PC 앱 코드 인증 및 사용 시작** 중심으로 재구성하였다.
- **사용 시간 연장 신청** 흐름은 **자동 로그아웃**까지 포함하는 형태로 확장하였다.
- 관리자 관련 흐름과 자동 종료 흐름은 유지하되, 현재 구조에 맞게 정리하였다.

즉, 기존 6개 흐름을 현재는 **5개 흐름**으로 재정리하였다.

## 6. 최종 구조의 장점

### 6.1 요구사항 변화 반영
모바일 앱 코드 표시, PC 앱 코드 입력, 미연장 시 자동 로그아웃 같은 최신 요구사항이 활동 다이어그램에 반영되었다.

### 6.2 PC 앱 담당 범위 명확화
모바일 앱 내부 동작을 분리하고, PC 앱이 실제로 담당하는 코드 인증 반영, 화면 전환, 연장 처리, 관리자 기능, 자동 종료에 집중하였다.

### 6.3 가독성 향상
일반 사용자 로그인 흐름을 제거하고, 핵심 기능별로 다이어그램을 분리하여 읽기 쉽고 이해하기 쉬운 구조로 정리하였다.

### 6.4 구현 연결성 강화
현재 클래스 다이어그램의 `PcMainScreen`, `PcUsageManager`, `AdminLoginScreen`, `AdminAuthManager`, `AdminDashboardScreen`, `AdminManager` 구조와 자연스럽게 연결된다.

최종 PC 앱 활동 다이어그램은 변경된 요구사항을 반영하여, 일반 사용자 PC 로그인 제거와 코드 인증 방식 변경, 자동 로그아웃 정책을 중심으로 새롭게 정리되었다.

정리하면 다음과 같다.

- 일반 사용자 흐름은 **PC 앱 코드 인증 및 사용 시작**으로 정리하였다.
- 연장 관련 흐름은 **사용 시간 연장 신청 및 자동 로그아웃**으로 확장하였다.
- 관리자 흐름은 **관리자 로그인**, **관리자 조회 / 제한 / 강제 종료**로 유지하였다.
- 종료 흐름은 **자동 종료**로 독립적으로 표현하였다.

이 구조를 기준으로 PC 앱 구현을 진행하면, 이후 실제 React 화면 구성과 상태 흐름 설계에도 일관되게 연결할 수 있다.
